# PRD: AI画像生成 × SVD アニメーション作品

## 概要

Ollama (LLM) + Stable Diffusion 1.5 + Stable Video Diffusion (SVD) を組み合わせ、
4枚の「はがきサイズ」イラストを生成し、それぞれをアニメーション化して
スマホ縦型（9:16）の90秒動画を自動生成するシステム。

---

## 1. 目的

- AIでストーリーから画像・動画を自動生成するパイプラインの構築
- スマホ向け縦型動画（90秒）を出力
- 4シーンのイラストが動き、物語が進行する体験

---

## 2. 技術構成

| コンポーネント | 技術 | 役割 |
|---------------|------|------|
| ストーリー生成 | Ollama (Python) | シナリオ・プロンプト生成 |
| 画像生成 | Stable Diffusion 1.5 | はがきサイズのイラスト生成 |
| 動画生成 | Stable Video Diffusion (SVD) | 静止画→アニメーション |
| 動画編集 | FFmpeg | 結合・リサイズ・エンコード |

---

| 項目 | 仕様 |

## 5. 各ステップ詳細

### Step 1: ストーリー生成 (Ollama)

```
入力: テーマ（例: 「夏の冒険」「不思議な森」）
出力: 4シーンのプロンプト + ナレーション
```

**プロンプト設計:**
```
あなたはアニメーションの脚本家です。
以下のテーマで4シーンの物語を作成してください。

テーマ: {theme}

各シーンについて以下を出力:
1. シーンタイトル
2. 画像生成プロンプト（英語、 Stable Diffusion形式）
3. アニメーションの動き説明
4. ナレーション（日本語、22.5秒分）
```

### Step 2: 画像生成 (SD 1.5)

```
入力: Step1のプロンプト×4
出力: はがきサイズ画像 (1024×576) ×4枚
```

**設定:**
- モデル: `runwayml/stable-diffusion-v1-5`
- サンプラー: Euler a
- ステップ数: 30
- CFG: 7.5
- サイズ: 1024×576

### Step 3: 動画生成 (SVD)

```
入力: はがき画像×4
出力: アニメーションクリップ×4 (25フレーム)
```

**設定:**
- モデル: `stabilityai/stable-video-diffusion-img2vid-xt`
- フレーム数: 25
- モーションベクトル: 127 (デフォルト)
- ガイドスケール: 3.0

### Step 4: 動画編集 (FFmpeg)

```
入力: アニメーションクリップ×4 + ナレーション音声
出力: output.mp4 (90秒)
```

**処理内容:**
1. 各クリップを1080×1920にリサイズ・回転
2. 24fpsに変換
3. クロスフェードで結合
4. ナレーション音声を同期
5. H.264でエンコード
